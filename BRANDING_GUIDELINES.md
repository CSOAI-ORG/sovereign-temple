# MEOKCLAW Brand Guidelines v1.0
## Sovereign AI Operating System

---

## 1. BRAND ESSENCE

### Mission
Enable every individual and organization to run sovereign multi-model intelligence on their own hardware — without extraction, surveillance, or rent.

### Positioning
MEOKCLAW is not "another AI chatbot." It is an operating system for decentralized intelligence. We are to AI what Linux is to operating systems: the open, ownable alternative to proprietary monopoly.

### Core Beliefs
1. **Sovereignty**: Your data, your models, your rules. No exceptions.
2. **Consensus**: No single model is trustworthy. Democracy applies to intelligence.
3. **Mesh**: Intelligence is stronger when distributed, not centralized.
4. **Anti-Rent**: Software should be owned, not subscribed to indefinitely.

---

## 2. VISUAL IDENTITY

### Logo Concept
An abstract neural mesh rendered as a 3D lattice of 8 interconnected nodes. The lattice forms the shape of a claw — protective, sharp, precise. In negative space, the nodes suggest a brain hemisphere split.

**Do not**: Use generic robot heads, chat bubbles, or cloud icons.

### Color Palette

| Token | Hex | Usage |
|-------|-----|-------|
| **Sovereign Teal** | `#00d4aa` | Primary accent, CTAs, active states, links |
| **Void Black** | `#0a0a0f` | Background, dark mode base |
| **Node Gray** | `#151520` | Cards, panels, secondary surfaces |
| **Mesh Gray** | `#1a1a2e` | Borders, dividers, inactive states |
| **Signal White** | `#e0e0e0` | Primary text |
| **Warning Amber** | `#f59e0b` | Errors, cautions, timeouts |
| **Cloud Red** | `#ef4444` | Only for "cloud/locked" comparison visuals |

### Gradient Accents
- Primary: `linear-gradient(135deg, #00d4aa 0%, #00a3ff 100%)` — hero backgrounds, highlights
- Mesh: `linear-gradient(180deg, #0a0a0f 0%, #151520 100%)` — panels, cards

### Typography
- **UI**: `system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- **Code**: `ui-monospace, SFMono-Regular, 'SF Mono', Menlo, monospace`
- **Headlines**: 700 weight, tight tracking (-0.02em), line-height 1.1
- **Body**: 400 weight, normal tracking, line-height 1.6
- **Monospace**: 400 weight, used for API endpoints, model names, code blocks

## 6. PHOTOGRAPHY & ILLUSTRATION

### Style
- Dark, cinematic tech photography
- Macro shots of silicon, fiber optics, neural patterns
- No stock photos of people "using laptops"
- Abstract 3D renders of mesh topologies
- Glitch-art aesthetics for "cloud vs local" comparisons

### Prohibited
- Smiling people pointing at screens
- Generic cloud imagery (unless mocking it)
- Bright, SaaS-style pastel gradients
- Cartoon robots

---

## 7. APPLICATION

### Web UI
- Dark mode only (for now)
- Teal accents on interactive elements
- Monospace for model names and API paths
- Subtle node-connection animations on load

### Mobile Apps
- Same dark palette
- Teal pulsing indicator when mesh is active
- Haptic feedback on message send

### Presentations
- Black background, teal accents, white text
- No bullet points longer than 12 words
- Every slide answers: "So what?"

### Social Media
- Twitter/X: Short, punchy, code snippets, screenshots
- GitHub: Clean README, architecture diagrams, benchmark tables
- YouTube: Screen recordings with minimal editing, raw authenticity

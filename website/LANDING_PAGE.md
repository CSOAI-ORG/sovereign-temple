## HOW IT WORKS

### Step 1: Install
`curl -fsSL meok.ai/install | bash`
One command. Ollama, models, APIs — everything.

### Step 2: Connect
Your MacBook M4 discovers your MacBook Air M2 automatically. Add a Vast.ai GPU for heavy lifting. Or don't — local works perfectly.

### Step 3: Ask
Type or speak. The mesh routes to the best model for your question. Code → Owl Alpha. Reasoning → DeepSeek V4. Fast → local Qwen3. Vision → Gemma 4.

### Step 4: Verify
Multiple models answer independently. The council votes. You get consensus, not hallucination.

## FAQ

**Q: Is this really free?**
A: MEOKLOCAL is MIT-licensed and free forever. You pay for your own electricity, not our rent.

**Q: Do I need a powerful computer?**
A: No. MEOKCLAW runs on a MacBook Air M2 (8GB) with Qwen3-4B. Add an M4 or cloud GPU for heavier models.

**Q: What models are supported?**
A: Any Ollama-compatible model, any OpenRouter model, any OpenAI-compatible API. Currently tested: Qwen3, Gemma 4, Llama 3, DeepSeek, Owl Alpha, Nemotron.

**Q: Is my data private?**
A: By default, 100%. Local models process everything on your hardware. Cloud models are only used if you explicitly enable them and route to them.

**Q: How is this different from Ollama?**
A: Ollama runs single models. MEOKCLAW orchestrates multiple models into a mesh with consensus voting, speculative decoding, and automatic routing.

**Q: Can I use this for my company?**
A: Yes. MEOKLOCAL is MIT-licensed. MEOKCLOUD Enterprise adds compliance, SSO, and support.

**Q: Will you add Windows/Linux support?**
A: Yes. macOS first (our dev environment), Windows and Linux coming Q3 2026.

**Q: How do I contribute?**
A: GitHub: github.com/nicholastempleman/meokclaw. PRs welcome.

**Q: Who built this?**
A: Nicholas Templeman, a solo founder in the UK building sovereign infrastructure because the alternative is unacceptable.

**Q: What's the roadmap?**
A: Q2: Mobile apps + Windows bridge. Q3: Enterprise compliance + custom models. Q4: Community mesh (decentralized nodes worldwide).
